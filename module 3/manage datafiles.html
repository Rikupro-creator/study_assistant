1. Resizing Data Files

ALTER DATABASE DATAFILE 'full_path' RESIZE new_size;

Alternative: Use data file number instead of full path

Use Cases:

  • Fix overestimated tablespace size
  • Increase size instead of adding new file
  • Prepare for large data loading

Example:

Resize data file to 500MB:

ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf' RESIZE 500M;

Using file number:

ALTER DATABASE DATAFILE 5 RESIZE 1G;
Note: Only ALTER DATABASE statement works for resizing data files.

2. Altering Data File Availability

ALTER DATABASE DATAFILE 'full_path' OFFLINE;

Alternative tablespace syntax:

ALTER TABLESPACE 'tablespace_name' DATAFILE 'full_path' OFFLINE;

FOR DROP Option

ALTER DATABASE DATAFILE 'full_path' OFFLINE FOR DROP;
  • Used in NOARCHIVELOG mode
  • Cannot bring file back online after using this option
  • Does not actually drop the file

Example:

Take a data file offline for maintenance:

ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users02.dbf' OFFLINE;

Bring it back online when done:

ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users02.dbf' ONLINE;

3. Renaming/Relocating Data Files

Online Method

ALTER DATABASE MOVE DATAFILE 'current_path' TO 'new_path';

Keywords:

  • REUSE: Overwrites existing file with same name
  • KEEP: Copies file (retains original), default behavior moves file

Offline Method

  1. Take data file offline cleanly
  2. Relocate/rename file externally (OS commands)
  3. Update database with:
    ALTER DATABASE RENAME FILE 'old_path' TO 'new_path';
  4. Recover data file: RECOVER DATAFILE 'new_path';
  5. Backup database/tablespace
  6. Bring online: ALTER DATABASE DATAFILE 'new_path' ONLINE;

Example (Online):

ALTER DATABASE MOVE DATAFILE '/old_location/users.dbf' TO '/new_location/users.dbf' REUSE;

4. Dropping Data Files

ALTER TABLESPACE 'tablespace_name' DROP DATAFILE 'full_path';

Alternative: Use data file number

For Temporary Files:

ALTER DATABASE TEMPFILE 'full_path' DROP;

Restrictions:

  • File must be empty (no segments)
  • Database must be open
  • Cannot drop last data file in tablespace
Note: No ALTER DATABASE statement exists for dropping data files, only ALTER TABLESPACE.

Example:

ALTER TABLESPACE USERS DROP DATAFILE '/u01/app/oracle/oradata/ORCL/users03.dbf';

Key Concepts

Relationship

One-to-many (tablespace to data files)

Data Storage

User data is saved in data files

Online Operations

Many operations can be performed while database is active

Consistency

Required for offline operations and recovery

Scenario:

You're managing an Oracle database with a tablespace called "SALES_DATA". This tablespace has a data file located at /u01/app/oracle/oradata/ORCL/sales01.dbf that is currently 200MB in size.

Task 1: You need to resize this data file to 500MB to accommodate upcoming sales data. Write the command.

Task 2: After resizing, you need to move this data file to a new storage location at /u02/app/oracle/oradata/ORCL/sales01.dbf while keeping the database operational. Write the command.

Task 3: You discover there's an unused data file at /u01/app/oracle/oradata/ORCL/sales_old.dbf in the SALES_DATA tablespace that you want to remove. Write the command to drop it.
